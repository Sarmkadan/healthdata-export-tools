# Security Policy

## Supported Versions

We currently support the following versions of healthdata-export-tools with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of healthdata-export-tools seriously. If you discover a security vulnerability within this project, please responsibly disclose it to us.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them using one of the following methods:

1. **GitHub Private Advisory:** Use GitHub's built-in private vulnerability reporting feature on this repository.
2. **Email:** Contact us directly at **rutova2@gmail.com**.

### What to include in your report
Please provide as much information as possible to help us investigate and reproduce the issue, including:
- A description of the vulnerability.
- Steps to reproduce the issue.
- Potential impact of the vulnerability.
- Any suggested fixes or mitigations (optional).

### Response Time
We will acknowledge receipt of your vulnerability report within **48 hours**. We will then investigate the issue and work on a fix as quickly as possible. Once the issue is resolved, we will publish a security advisory and notify you.

Thank you for helping to keep healthdata-export-tools safe!
